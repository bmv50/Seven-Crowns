# Ленивая симуляция мира (Catch-up)

Чтобы не тратить CPU на тик каждой комнаты каждую секунду, мир можно перевести в
«ленивый» режим: тикают только активные комнаты (где есть игроки + соседние),
а спящие «догоняются» при входе игрока — прошедшее время применяется одним махом
(респавн мобов по абсолютному таймеру, рост потребностей NPC). Идея из `TeleMud/`.

## Как включить
По умолчанию **выключено** (мир тикает как раньше). Включается переменной окружения,
как RULES_V2 / NPC_AI / WILD_ZONES:
```powershell
$env:LAZY_SIM = "1"      # или строка LAZY_SIM=1 в .env
python -m bot.main
```

## Что меняется
- Бой и «жизнь» NPC обрабатываются только в активных комнатах (игроки + соседи
  1-го радиуса). Бой и так возможен лишь там, где есть игроки, — поведение для
  игрока не меняется, экономится только холостая работа по пустым комнатам.
- Респавн становится «ленивым»: моб в дальней пустой комнате возрождается не по
  фоновому таймеру, а в момент, когда игрок туда приходит (по тому же интервалу
  `respawn`, расчёт по абсолютному времени — корректно при любой паузе).

Награды, лут, баланс — не затронуты.

## Тонкая настройка (`engine/catchup.py`)
- `COARSE_COMBAT/COARSE_NPC/COARSE_RESOURCE` — крупные кванты времени (10с/5мин/30мин)
  для разбивки ΔT (как в референсе).
- `active_set()` — радиус активации (сейчас игроки + соседи 1-го уровня).

## Проверка
```powershell
python test_catchup.py     # тесты ленивой симуляции
python test_engine.py      # все тесты движка (278 ✅)
```
