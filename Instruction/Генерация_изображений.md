# Генерация изображений предметов (ComfyUI / Flux)

Инструкция для тебя — как генерировать арт предметов и как он попадает в игру.

## Что нужно
- **ComfyUI запущен** на этой машине (по умолчанию `http://127.0.0.1:8188`), с моделью Flux
  (узлы из `workflow_final.json`: `flux-2-klein-9b-fp8`, CLIP `qwen_3_8b`, VAE `flux2-vae`).
- **Python** (тот же, которым запускаешь бота). Доп. библиотек не нужно — скрипт на stdlib.
- В корне проекта лежит **`workflow_final.json`** (API-формат ComfyUI).

## Запуск (PowerShell, из корня проекта)
```powershell
# 1) сначала тест на 2 предметах — проверить связь с ComfyUI
python scripts/gen_item_images.py --limit 2

# 2) если ок — все именованные предметы (оружие/броня/аксессуары/зелья) + руны (~94)
python scripts/gen_item_images.py

# полезные опции:
python scripts/gen_item_images.py --type weapon      # только оружие (armor/accessory/consumable/rune)
python scripts/gen_item_images.py --only-missing     # пропустить уже готовые (докинуть недостающие)
python scripts/gen_item_images.py --include-gen      # ВЕСЬ каталог, включая тиры g_* (~255, долго)
python scripts/gen_item_images.py --size 1024        # размер картинки (по умолчанию 1024×1024)
python scripts/gen_item_images.py --host 127.0.0.1:8188
```

## Что происходит под капотом
1. Скрипт берёт каждый предмет из `ITEMS`, строит **промпт** из русского названия + типа + описания
   (CLIP Qwen мультиязычный, русский понимает).
2. Подставляет промпт в узел `7` (CLIPTextEncode), рандомит сид (узел `11`), ставит квадрат (узел `8`).
3. Отправляет в `POST /prompt`, ждёт `GET /history/<id>`, забирает картинку через `/view`.
4. Кладёт в **`images/items/<ключ_предмета>.png`** (например `ржавый_меч.png`).

## Как картинки попадают в игру
Ничего вручную делать не нужно. При показе карточки предмета бот (`bot/item_images.py`)
**автоматически** накладывает на твой арт цветную рамку редкости:
🟢 редкая · 🔵 эпическая · 🟣 мифическая · 🟡 легендарная · 🔴 божественная (⚪ простая — без цвета).
Готовые рамки кэшируются в `images/items_cache/`. Если ты **перегенерил** арт (файл новее кэша) —
бот пересоберёт рамку сам.

## Стиль и тонкая настройка
Всё в `scripts/gen_item_images.py`:
- **`STYLE`** — общий стиль (сейчас: один предмет по центру, тёмный нейтральный фон, чистая иконка,
  без текста/рамки). Меняй тут, если нужен другой арт-дирекшн (например, белый фон).
- **`TYPE_RU`** — как описывается каждый тип предмета в промпте.
- **`RUNE_LOOK`** — руна = гранёный **кристалл-самоцвет** (НЕ камень) с цветом под стат и
  **иконой навыка** внутри:
  - 🔴 Сила — рубиново-красный кристалл, символ «сжатый кулак»
  - 🟢 Ловкость — изумрудно-зелёный, «летящая стрела»
  - 🔵 Интеллект — сапфирово-синий, «раскрытая книга»
  - 🟣 Дух — аметистово-фиолетовый, «язык пламени»
  - 🟡 Защита — золотисто-янтарный, «рыцарский щит»
  - ⚪ Крит — прозрачно-алмазный, «скрещённые мечи»

  В каждом промпте есть негативы `НЕ камень, без мха, без растительности, без трещин` (`RUNE_NEG`).
  Большая руна — крупный многогранник с ярким свечением, малая — мягче.
  Хочешь вместо иконы букву (S=сила, D=ловкость…)? Замени `icon` в `RUNE_LOOK` на нужную букву.
- **`build_prompt(key)`** — здесь собирается финальный промпт; правь под себя.

Совет: фон лучше держать **тёмным и нейтральным** — поверх него красиво ложится цветная рамка
редкости. Если фон сделать ярким/цветным, рамка будет «спорить» с ним.

## Если что-то не так
- **`Не найден workflow_final.json`** — положи файл в корень проекта.
- **`нет prompt_id` / ошибка соединения** — ComfyUI не запущен или другой порт; проверь `--host`.
- **`таймаут ожидания`** — генерация дольше ~3 мин (тяжёлая модель/очередь). Увеличь лимит ожидания
  в `generate_one` (цикл `range(180)`), либо уменьши `steps` в workflow.
- **Картинки «пустые»/одинаковые** — уточни промпт в `build_prompt`/`STYLE`. Сид рандомный, так что
  повторный запуск того же предмета даст другой вариант.

## Перегенерация конкретного предмета
Просто удали его файл из `images/items/` и запусти с `--only-missing`, либо сгенерируй точечно через
`--type` + `--limit`. Бот подхватит новый арт автоматически (по дате файла).

---

## Мобы и локации (тот же ComfyUI Flux)
Скрипт `scripts/gen_world_images.py` генерирует арт **мобов** и **локаций** по тому же
`workflow_final.json`:
```powershell
python scripts/gen_world_images.py --type mob                 # все мобы → images/mobs/
python scripts/gen_world_images.py --type room                # все локации → images/rooms/
python scripts/gen_world_images.py --type room --include-wild  # + процедурные дикие зоны
python scripts/gen_world_images.py --type mob --only-missing   # докинуть недостающие
python scripts/gen_world_images.py --type room --limit 5       # тест на 5
```
Мобы рисуются квадратом 1024² (портрет существа), локации — ландшафтом 1216×704
(атмосферная сцена без персонажей). Промпты строятся из `mobs.yaml` / `world.yaml`
(имя, уровень, тип урона / биом, зона, описание). Бот сам подхватит новые png по дате.

### Умные промпты комнат (обновление)
`scripts/gen_world_images.py --type room` теперь строит промпт с учётом:
- **Ключевых NPC** комнаты (до 2): имя + визуальная роль (страж в доспехе, торговец за
  прилавком, жрец, наставник…). Берётся из `WORLD[room].npc` + `NPCS`.
- **Освещения по типу локации** (`_LIGHT` в скрипте): подземелья/норы/штольни/бездна —
  очень тёмно; болота — зелёный сумрак; леса — рассеянный свет сквозь кроны; гавань/город
  у воды — морской сумрак; пепельные пустоши — багровое зарево; перевалы — пасмурно;
  храмы/Чертоги Рассвета — тёплый свет витражей; города — пасмурный день с факелами.
- **Расового стиля** столицы (`_RACE_ZONE`): автоматически по зоне старта расы
  (человек/орк/эльф/гном/гоблин) добавляется архитектура и палитра.

Хочешь подправить — три места в скрипте: `_LIGHT` (освещение), `_RACE_ZONE` тема рас,
`_ROLE_VIS` (как роль NPC выглядит). Команда запуска не меняется.

### Без текста на картинках (Flux)
Важно: у Flux в этом воркфлоу НЕТ негативного промпта, поэтому слова «без текста,
без подписей» в позитивном промпте работают наоборот — провоцируют модель рисовать
(кривой) текст. Поэтому:
- Из всех стилей (`STYLE_COMMON`, `STYLE`, `ROOM_STYLE`, `MOB_STYLE`) убраны фразы
  «без текста / без подписей / без рамок».
- Из промптов **комнат** убраны названия локаций/зон и **имена NPC** — остаются только
  описание сцены, расовая тема, освещение и визуальные роли («вооружённый страж»,
  «торговец за прилавком»).
- Промпты **мобов** больше не содержат имя — существо описывается по типу
  (`_MOB_KIND`: нежить, демон, дракон, голем, паук, зверь, грызун, слизень, гоблиноид,
  латный воин…) + угроза по уровню + стихия по типу урона.

Если где-то текст всё равно проскочит — скинь пример, точечно подправлю описание.

### Перегенерация отдельных комнат
Два способа переснять конкретную локацию (id комнаты = ключ в `world.yaml`, напр. `inn_room`, `temple`):

```powershell
# 1) точечно по id (можно несколько через запятую) — самый удобный
python scripts/gen_world_images.py --type room --ids inn_room
python scripts/gen_world_images.py --type room --ids inn_room,temple,harbor

# 2) удалить файл и догенерировать недостающие
del images\rooms\inn_room.png
python scripts/gen_world_images.py --type room --only-missing
```
Сид случайный, поэтому повторный запуск того же id даёт новый вариант картинки —
гоняй `--ids <id>`, пока не понравится. Бот подхватывает новый png автоматически по дате файла
(старый кэш-рамку, если есть, пересоберёт сам). Узнать id комнаты можно по `world*.yaml`
или по уже сгенерированному файлу в `images/rooms/<id>.png`.
