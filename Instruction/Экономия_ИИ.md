# Контроль стоимости ИИ-реплик NPC

Чтобы живые реплики NPC (DeepSeek) не жгли бюджет и токены, добавлены три механизма.
Они работают **автоматически**, как только включён ИИ-провайдер (`AI_PROVIDER=deepseek`
+ ключ) — отдельный флаг не нужен. Адаптировано из референс-движка `TeleMud/`.

## Что делает
1. **Кэш похожих вопросов.** Если игрок спрашивает почти то же самое, что уже спрашивали
   этого NPC, ответ берётся из кэша без обращения к модели (экономия запроса).
2. **Суточный лимит на NPC.** Один игрок не может бесконечно «доить» одного персонажа —
   после лимита запросов в сутки NPC отвечает шаблоном.
3. **Тайм-аут диалога.** После паузы беседа сжимается в одно короткое «воспоминание»
   (локально, без лишнего запроса) и подставляется в следующий разговор — NPC «помнит»,
   о чём вы говорили, но контекст не растёт бесконечно.

## Настройка (в `.env`)
```ini
AI_DAILY_PER_NPC=30     # сколько запросов к модели на (игрок, NPC) в сутки
AI_CACHE_SIM=0.9        # порог сходства вопросов для кэша (0..1; выше = строже)
AI_DIALOG_TIMEOUT=300   # пауза в секундах, после которой диалог «сворачивается»
```
Значения по умолчанию указаны выше — можно не задавать.

## Действия NPC (задел)
Модель теперь может предложить игровое действие (например, выдать доступный квест),
вернув в конце ответа JSON. Движок проверяет его по реальному состоянию игры и отбрасывает
выдуманное. Пока действие парсится и валидируется, но кнопку под репликой в боте ещё не
рисуем — это следующий шаг (скажи, если подключить).
