# 🧱 Схема контента и шаблоны (для авторов и субагентов)

> Строгий формат YAML-фрагментов. Любой регион описывается **пятью блоками**:
> WORLD / MOBS / ITEMS / QUESTS / NPCS. Каждый блок — валидный YAML, который
> дописывается в одноимённый файл `data/*.yaml`. Держись форматов дословно,
> иначе валидатор (`python -m engine.content`) упадёт.

Перед работой прочти `WORLD_BIBLE.md` (лор, фракции, регионы, уровни важности).

---

## Общие правила

- **Комнаты/зоны** — id латиницей snake_case (`iron_keep_gate`). **Мобы,
  предметы, NPC, квесты** — id как в существующих файлах (мобы/предметы —
  кириллица snake_case: `шахтный_страж`; квесты — латиница: `iron_militia`).
- **Не переопределяй** существующие id. Уникальность — глобальная по файлу.
- **Выходы двусторонние:** если новая комната цепляется к существующей
  (точка входа указана в задании), пропиши выход И обратно из неё. Все цели
  выходов/спавнов/лута/наград — существующие id или из твоего же фрагмента.
- **Баланс:** hp/atk/xp/gold мобов держи в коридоре уровней региона (см.
  таблицу регионов в библии). Пиши «базовые» hp как у текущих мобов — движок
  сам умножит на глобальный `HP_SCALE`.
- **Тексты** русские, атмосферные; описание комнаты 2–4 предложения.
- Каждый новый предмет должен иметь применение: где падает/покупается/
  крафтится/выдаётся в награду И зачем нужен.

## Блок WORLD (комнаты) → `world.yaml`

```yaml
iron_keep_gate:
  name: "Ворота Железного Острога"
  zone: "Железный Острог"          # человекочитаемое имя региона (как в библии)
  desc: >
    Окованные железом ворота шахтёрского города. Со стен на туман смотрят
    арбалетчики, факелы чадят в сырости. За воротами слышен стук молотов.
  exits: { запад: mine_deep, север: iron_keep_market }   # двусторонние!
  npc: [страж_острога]             # id из блока NPCS (или существующие)
  spawns: []                       # id мобов (пусто, если безопасно)
  items: []                        # id предметов на земле
  # опционально:
  # rest: true                     # комната-отдых (полное восстановление HP/MP)
```

Правила: `zone` группирует регион; у безопасных городских комнат `spawns: []`.
Точка стыковки с миром задаётся в задании субагенту (напр. «вход из `mine_deep`
по выходу `восток`» — тогда у `mine_deep` уже есть `восток: iron_keep_gate`,
а у твоей комнаты — `запад: mine_deep`).

## Блок MOBS (враги) → `mobs.yaml`

```yaml
шахтный_страж:
  name: "Шахтный страж"
  emoji: "⛏"
  level: 6
  hp: 70            # базовое (движок ×HP_SCALE). Ориентир: lvl×~12
  atk: 14           # ориентир: lvl×~2.2
  defense: 4
  xp: 60            # ориентир: lvl×~10
  gold: 28
  respawn: 90       # сек; мини-боссы 150–240; боссы 99999
  tick_speed: 4     # как часто бьёт, сек (быстрые 3, медленные 5)
  loot: [["железная_руда", 0.4], ["осколок_голема", 0.3]]
  # опц.: skills: ["power_strike"]   # только существующие скиллы
  # опц.: boss: true                 # для боссов/мини-боссов региона
```

## Блок ITEMS (предметы) → `items.yaml`

```yaml
острожный_клинок:
  name: "Острожный клинок"
  emoji: "⚔️"
  type: weapon          # weapon|armor|accessory|consumable|material|quest
  slot: weapon          # weapon|armor|accessory (для экипировки)
  bonus: { atk: 10, str: 1 }
  price: 100            # для покупки/продажи (продажа = 60% цены)
  # class_req: [warrior]   # опц. ограничение класса
  desc: "+10 атаки, +1 Силы. Ковка Острога."

# материал/лут одной строкой (flow-стиль) — ОБЯЗАТЕЛЕН ПРОБЕЛ после двоеточия:
руда_тумана: { name: "Руда тумана", emoji: "🌫", type: material, price: 16, desc: "Сырьё, гудящее эхом." }
```

⚠️ Частая ошибка: `ключ:{...}` без пробела ломает YAML. Пиши `ключ: { ... }`.

## Блок QUESTS (квесты) → `quests.yaml`

```yaml
iron_militia:
  name: "Ополчение Острога"
  giver: страж_острога        # id NPC из блока NPCS
  turn_in: страж_острога
  desc: >
    Стража просит проредить шахтных тварей, что лезут к стенам. Убей
    5 шахтных стражей в штольнях.
  objective: { type: kill, mob: шахтный_страж, count: 5 }   # kill|collect
  reward: { xp: 120, gold: 70, items: [острожный_клинок] }
  # requires: ore_supply       # опц. предыдущий квест цепочки
```

`collect`-цель: `objective: { type: collect, item: руда_тумана, count: 3 }`.

## Блок NPCS (персонажи) → `npcs.yaml`

```yaml
страж_острога:
  name: "Капитан стражи Бьёрн"
  emoji: "🛡"
  role: questgiver            # questgiver/vendor/priest/guard/faction_leader/
                              # trainer/gossip/boss/entity
  importance: named           # ambient|common|named|key (→ модель ИИ)
  faction: volnye_rudokopy    # id из factions.yaml
  persona: >
    Суровый капитан стражи Острога, не доверяет чужакам, но ценит дело.
    Говорит коротко, по-военному. Предан гильдии рудокопов.
  knowledge: [острог, стража, шахтные_твари, вольные_рудокопы]
  lines:
    - "🛡 *Капитан Бьёрн:* «Чужак? Покажи делом, что не из туманного отродья.»"
```

Дополнительно: каждого NPC, стоящего в комнате, впиши в её `npc:`-список (WORLD).
Лидеры фракций — `importance: key`; рядовые стражи/торговцы — `common`;
безымянная массовка — `ambient`.

## Формат сдачи фрагмента (для субагента)

Верни РОВНО пять YAML-блоков под заголовками, без пояснений вне YAML:

```
### WORLD
<...валидный yaml...>
### MOBS
<...>
### ITEMS
<...>
### QUESTS
<...>
### NPCS
<...>
```

Если блок пуст — напиши `# (нет)`. Все стыковки с существующим миром —
по точке входа из задания. Никаких новых механик/полей вне этой схемы.
