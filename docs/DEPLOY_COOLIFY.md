## Часть 1. Публикация на GitHub (сделать ПРАВИЛЬНО, история содержит секреты)

⚠️ **Начальные коммиты репозитория содержат реальные BOT_TOKEN и ключ DeepSeek
в `.env.example`.** Публиковать эту историю нельзя. Если ключи ещё не отозваны —
отзови СЕЙЧАС: BotFather → `/revoke`, platform.deepseek.com → удалить ключ.

### Рекомендуемый путь — свежая история (старая ценности не имеет: 2 коммита)

```powershell
# 0) РЕЗЕРВНАЯ КОПИЯ всей папки проекта (вместе с .git) — обязательно:
Compress-Archive -Path . -DestinationPath ..\seven_crowns_backup.zip

# 1) Дополнить .gitignore мусором, которому не место в публичном репо:
#    (добавь строки, если их нет)
#      ~/
#      graphify-out/
#      .sim_rules2_cache.json
#      backups/
#    TeleMud/ — чужой референс-проект на Go: если не уверен в его лицензии,
#    добавь и его в .gitignore (игре он не нужен, это исследовательский материал).

# 2) Убедиться, что .env НЕ отслеживается и секретов нет:
python scripts/check_secrets.py     # должно быть: находок нет

# 3) Свежая история:
Remove-Item -Recurse -Force .git
git init -b main
git add -A
git commit -m "Семь Корон: Telegram-MUD, бета-готовность (этапы 0-10)"

# 4) Создай на github.com ПУСТОЙ публичный репозиторий (без README), затем:
git remote add origin https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПО.git
git push -u origin main
```

Проверка после пуша: открой репо в браузере → `.env` отсутствует,
`.env.example` содержит только заглушки `ВСТАВЬ_СВОЙ_…`.

## Часть 2. Настройка в Coolify

Наш стек — `docker-compose.yml` в корне: сервисы `bot` (игра), `postgres`
(база, порт наружу закрыт), `backup` (ежедневный pg_dump в 04:00 UTC,
ротация 14 дней). У бота НЕТ HTTP-порта (long polling) — домен и прокси
в Coolify не нужны.

### Шаги

1. **Coolify → Projects → + New** → назови (напр. `seven-crowns`).
2. **+ New Resource → Docker Compose**. Источник:
   - публичный репо: «Public Repository» → вставь URL;
   - лучше: подключи **GitHub App** (Sources → GitHub) — даст автодеплой
     по пушу и работу с приватным репо, если позже закроешь его.
   Branch: `main`. Compose file: `docker-compose.yml` (корень).
3. **Environment Variables** (вкладка ресурса). Coolify передаёт их compose-у;
   наш compose читает `env_file: .env` — Coolify сам генерирует `.env` из
   переменных UI (проверь после первого деплоя в логах, что бот увидел токен;
   если нет — переключи в compose `env_file` на `environment`-блок или задай
   переменные как «Build/Runtime» в UI). Обязательные:

   | Переменная | Значение |
   |---|---|
   | BOT_TOKEN | токен от BotFather (НОВЫЙ, после revoke) |
   | DATABASE_URL | `postgresql://mud:СИЛЬНЫЙ_ПАРОЛЬ@postgres:5432/mud` — хост именно `postgres` |
   | POSTGRES_USER / POSTGRES_PASSWORD / POSTGRES_DB | `mud` / тот же сильный пароль / `mud` |
   | PROD | `1` |
   | ADMIN_IDS | твой Telegram uid (узнать: @userinfobot) |
   | AI_PROVIDER | `deepseek` (или `none` — игра полноценна на шаблонах) |
   | DEEPSEEK_API_KEY | новый ключ |
   | AI_DAILY_BUDGET_USD | напр. `3.0` — жёсткий дневной потолок трат LLM |

   Опционально: `LOG_JSON=1` (в образе уже включён), `GOD_INTERVAL=10800`,
   `DEEPSEEK_PRICE_IN/OUT` (уточни по актуальному прайсу DeepSeek),
   `PROXY_URL` (не нужен на зарубежном VPS).
4. **Deploy.** Первый билд ~2-4 мин. Миграции БД применяются автоматически
   при старте (все идемпотентны).
5. **Автодеплой:** при подключённом GitHub App — включи «Auto Deploy» в
   настройках ресурса: каждый `git push` = пересборка и перезапуск (мир
   переживает рестарт: снапшот в БД + финальный снапшот на shutdown).

### Проверка первого запуска

В логах контейнера `bot` (Coolify → ресурс → Logs) должны появиться:
`✅ PostgreSQL подключён…`, строки флагов мира, `🌩 Бог-оркестратор активен`,
`💾 Персистентность рантайма включена`, `⚔️ СЕМЬ КОРОН v3 запущена`.
Затем в Telegram: `/start` → создание персонажа; `/admin` → должно открыться
(если нет — проверь ADMIN_IDS). Healthcheck: контейнер `bot` в Coolify станет
Healthy через ~40-60 секунд (heartbeat).

## Часть 3. Эксплуатация

### Бэкапы (уже автоматические)
- Контейнер `backup` кладёт дампы в volume `./backups` ежедневно в 04:00 UTC,
  хранит 14 дней. Посмотреть: `docker exec <backup-контейнер> ls /backups`.
- **Раз в месяц — обязательный тест восстановления:**
  `docker exec <postgres-контейнер> sh /scripts/restore_test.sh` — или с хоста
  VPS: `bash scripts/restore_test.sh` при доступном DATABASE_URL. Exit 0 = дамп
  реально восстановим.
- Восстановление после аварии: остановить `bot` → `scripts/restore.sh`
  (спросит подтверждение YES) → запустить `bot`.
- Дополнительно рекомендуемое: скачивай последний дамп с VPS к себе раз в
  неделю (scp) — защита от потери самого сервера.

### Обновления игры
1. Правишь контент (data/*.yaml) или код локально.
2. `python -m engine.content` (валидатор) и `python run_tests.py` — зелёные.
3. `git add -A && git commit && git push` → Coolify задеплоит сам.
CI на GitHub (`.github/workflows/ci.yml`) прогонит секрет-скан и все тесты
на каждый пуш — не деплой при красном CI.

### Мониторинг
- **Coolify**: статус Healthy/Unhealthy контейнера bot (по heartbeat),
  рестарт-политика unless-stopped поднимет упавший процесс.
- **Внутри игры**: watchdog сам перезапускает упавшие воркеры (≤3/час) и
  шлёт алерт в Telegram всем ADMIN_IDS при серьёзных проблемах.
- **/admin → 💚 Health**: аптайм, онлайн, длительность тика, латентность БД,
  очереди, счётчик Telegram-429, последние ошибки.
- **/admin → 🤖 LLM**: расход бюджета за день; «⛔ исчерпан» = ИИ отключился
  сам до полуночи, игра работает на шаблонах (это штатно).
- Метрики беты: на VPS `docker exec <bot-контейнер> python scripts/funnel_report.py`
  — воронка /start→создание→первый квест, D1/D7, источники игроков.

### Типовые проблемы
| Симптом | Причина/действие |
|---|---|
| Бот Unhealthy, в логах ошибка BOT_TOKEN | заглушка/опечатка в переменной — исправь в Coolify UI, Redeploy |
| Процесс сразу выходит с кодом 1 | PROD=1 и БД недоступна (fail fast) — проверь POSTGRES_* и хост `postgres` в DATABASE_URL |
| NPC отвечают шаблонно | ИИ выключен: ключ/бюджет/тумблер — смотри /admin → LLM |
| Пуши приходят ночью | игрок не выставил пояс: Настройки → 🔔 → 🕐 Часовой пояс |
| Игрок токсичен | /admin → 👤 Игрок → мут/бан (всё журналируется) |
| Нужно заморозить торговлю | /admin → 🛑 Торговля |

### Регулярный ритм владельца (15 минут в день на бете)
1. Утро: /admin Health + LLM-бюджет глазами.
2. Раз в 2-3 дня: `funnel_report.py`, сверка с воротами
   (character_created ≥75% от /start, D1 ≥35-40%, D7 ≥15-20%).
3. Раз в неделю: скачать бэкап, прочитать жалобы (audit_log action='report'
   приходят админам в Telegram сразу).
4. Раз в месяц: restore_test.sh.
