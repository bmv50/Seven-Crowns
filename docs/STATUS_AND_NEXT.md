## Как поднять проект с нуля (онбординг)

Подробно — в `SETUP_WSL2.md`. Кратко (в терминале WSL2 Ubuntu):
```bash
# 1. Postgres
sudo service postgresql start
# (один раз создать БД: см. init_db.sql)

# 2. Окружение
cd ~/mud
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 3. Конфиг
cp .env.example .env   # вписать BOT_TOKEN, при блокировке Telegram — PROXY_URL

# 4. Проверка + запуск
python -m engine.content   # должно: ✅ Контент валиден...
python -m bot.main         # должно: ✅ PostgreSQL подключён... ⚔️ ...запущена
```
В Telegram `/start` → выбор расы.

**Важные нюансы окружения:**
- Команды — внутри **WSL2**, не PowerShell.
- Проект — в Linux-разделе (`~/mud`), не в `/mnt/c/`.
- Postgres в WSL2 не стартует сам после перезагрузки.
- В регионе разработки Telegram API блокируется → нужен `PROXY_URL` в `.env`.

## Контекст для ИИ-ассистента, продолжающего работу

Если ты (Claude в Cowork или другой ассистент) продолжаешь этот проект — важное:

1. **Прочитай документы в `docs/` в порядке:** PRODUCT_OVERVIEW → WHAT_WE_BUILT → ARCHITECTURE → ROADMAP → этот файл. Плюс `SPEC_AI_World.md` в корне для ИИ-видения.
2. **Держись принципов из ARCHITECTURE.md:** движок — арбитр, ИИ — советник; деградация без ИИ; контент в YAML + валидатор; слоистость (engine не знает про Telegram).
3. **Перед изменениями контента** — прогоняй `python -m engine.content` (валидатор целостности).
4. **Тестируй движок без Telegram** — он от него не зависит, можно гонять функции `engine/` напрямую (так тестировались классы, бой, квесты).
5. **Не зашивай модель/провайдера в код** — через абстракцию `ai/` и `.env`.
6. **Характеристики персонажа** меняются только в `character.py` (всё вычисляется из класса/расы/уровня/экипировки).
7. **Не тащи внешние «комбайны»** — ставь целевые библиотеки под конкретную задачу.
