## Фаза 0 — Безопасность и гигиена (1 день, делается ПЕРВОЙ)

1. **Отозвать и перевыпустить ключи.** В `.env.example` закоммичены реальные
   `BOT_TOKEN` и `DEEPSEEK_API_KEY`. Отозвать оба (BotFather → /revoke,
   platform.deepseek.com), вписать в `.env.example` заглушки `xxx`.
   Проверить `.gitignore` на `.env` (он есть, но `.env` лежит в рабочей папке —
   убедиться, что не закоммичен: `git log --all -- .env`).
2. **Git:** удалить висящий `.git/index.lock`, закоммитить всю текущую работу
   (сейчас 2 коммита при ~16k строк), настроить remote, включить CI
   (GitHub Actions: `python -m engine.content && python test_engine.py`).
3. **Актуализировать доки:** WHAT_WE_BUILT/STATUS_AND_NEXT устарели (пати/дуэли/гильдии
   давно подключены). Удалить или пересоздать фиктивный `world_runner_report.md`
   (в нём «1 день» в конфиге и «100-дневный тест» в выводах — артефакт генерации).

**Критерий:** ключи отозваны, `git status` чистый, CI зелёный.

## Фаза 1 — Затягиваемость: push-реактивация и возврат ✅ ВЫПОЛНЕНА 2026-07-04
*Итог: notify-система (94 теста), стрики+недельник (retention 140), сезонный
трек, коллекции, онбординг (47 тестов; 3 смерти вместо 12), рефералка.
Детали — CHANGELOG.md, спринты 1–3. Осталось из фазы: измерить D1/D7 на живых
игроках после деплоя (метрики — Фаза 4).*

### 1.1 Бот пишет первым (ядро фазы)
Новый модуль `engine/notify.py` + таблица `notifications` в Postgres:

- **События-триггеры:** сброс ежедневки (09:00 локали игрока), мировой босс заспавнился,
  лот на аукционе продан/перебит, кулдаун данжа истёк, «отдохнувший опыт» накопился до капа,
  конец сезона (за 24 ч и по факту с наградой), гильдейские события (вступление, вклад).
- **Анти-спам:** не более 2 push в сутки на игрока, тихие часы 23:00–09:00,
  настройки в меню (`ch.flags["notify"]`), деградация при 403 (юзер заблокировал бота →
  пометить и не слать).
- **Критично:** сейчас анонсы (мировой босс, победы) идут только по `chars` —
  это игроки в памяти процесса, т.е. онлайн с момента рестарта. Рассылку вести по
  списку uid из БД (`db.py` → `SELECT uid FROM characters WHERE ...`), с батчингом
  и rate-limit Telegram (≤30 msg/sec).

### 1.2 Циклы возврата
- **Стрик входов:** `ch.flags["streak"]` — бонус за подряд-дни (день 3: +10% опыта на день,
  день 7: сундук с рунами), сброс мягкий (заморозка стрика 1 раз в неделю).
- **Недельник:** по образцу `engine/daily.py` — `engine/weekly.py` c 3 задачами
  (данж, мировой босс, арена) и наградой-сундуком. Контент — `data/weekly.yaml`.
- **Сезонный трек:** `seasons.py` уже даёт лиги — добавить 10 «ступеней» наград за очки
  внутри сезона (бесплатный battle-pass), витрина прогресса в `/season`.
- **Коллекции:** `bestiary.py` уже пишет убийства — добавить награды за полноту
  («убил всех зверей леса → титул + питомец»).

### 1.3 Онбординг (сим показал 12 смертей до 10 уровня)
- Цепочка обучения из 5 шагов у наставника (двигаться → атаковать → скилл → зелье →
  сдать квест), каждый шаг — награда. Проверять прохождение флагами.
- Смерть до 5 уровня — без потери золота и с подсказкой («вернись, когда подлечишься»).
- Первая сессия должна давать: 3 уровень + первый лут + первый сданный квест за ≤10 минут.
  Проверить `sim_player.py` с профилем «новичок» (без зелий/скиллов): цель ≤3 смертей до 10 ур.

### 1.4 Социальные приглашения
- Telegram deep-link рефералка: `t.me/bot?start=ref_<uid>` → награда обоим на 5 уровне
  приглашённого. Хранить в `ch.flags["ref"]`.
- Кнопка «Позвать на босса» — шарит готовое сообщение с deep-link в комнату босса.

**Критерии фазы:** бот отправляет push по всем триггерам с анти-спамом; стрик и недельник
работают; sim-новичок проходит 1–10 с ≤3 смертями; рефералка начисляет награды.
Метрика после релиза: D1 ≥ 40%, D7 ≥ 20%.

## Фаза 3 — Новизна: ИИ-мир как главная фишка ✅ ВЫПОЛНЕНА 2026-07-07
*Итог: хроника мира (NPC сплетничают о реальных событиях сервера, игрокам —
экран «📜 Хроника»), бог-оркестратор (LLM запускает события из валидируемого
каталога 7 шт., летопись сезона, жёсткий бюджет, fallback без ИИ), ИИ-поручения
(динамические задачи с формульной наградой), долгая память NPC (npc_memories,
свежесть+лексика, слой готов под pgvector). 979 тестов. Осталось на живом
прогоне: god_worker с боевым DeepSeek-ключом — проверить при плейтесте.*

### 3.1 Бог-оркестратор (главное отличие от всех TG-игр)
`ai/god.py`: раз в N часов LLM получает сводку мира (онлайн, убийства боссов, флипы
территорий, экономика) и выбирает событие из **валидируемого каталога** (расширить
`data/events.yaml`: вторжение, ярмарка, аномалия, засуха лута, миграция мобов, пророчество).
Движок — арбитр: применяются только события из белого списка с параметрами в допустимых
границах (по образцу `ai/actions.py`). Событие анонсируется push-ем с ИИ-текстом.

### 3.2 NPC, которые знают, что происходит
- **Лента мира:** `engine/chronicle.py` — кольцевой лог событий (босс пал, зона перешла
  фракции, игрок X стал Легендой сезона). Последние 5 событий подмешивать в промпт NPC
  (`ai/npc_ai.py: WORLD_CANON` → динамическая часть). NPC сплетничают о реальных событиях.
- **Летопись недели:** в конце сезона LLM пишет «хронику» из лога (3 абзаца) —
  push всем + пост в канал игры. Игроки видят себя в истории мира.
- **pgvector-память** (спека готова): долгосрочные воспоминания именных NPC о игроках
  вместо усечённой строки в 300 символов. Таблица `npc_memories`, embedding DeepSeek/локальный.

### 3.3 ИИ-квесты по шаблонам
NPC может выдать **параметризованный** квест: LLM выбирает шаблон (kill/collect),
моба из своей зоны и обёртку-текст; движок валидирует (моб существует, уровень ±3 от
игрока, награда по формуле уровня). Это бесконечные «слухи-поручения» без риска
поломки баланса. Расширение `ai/actions.py` → `generate_errand`.

### 3.4 Лицо игры
- Генерация изображений уже есть (`gen_images*.py`) — добить: уникальные арты
  всех боссов и данжей, показывать при входе/спавне (карточки уже умеют фото).
- Сезонные темы: каждый сезон — модификатор мира (туман густеет: +нежить, −видимость
  на карте) + палитра/эмодзи в UI. Данные в `data/seasons_themes.yaml`.

**Критерии фазы:** мир меняется без разработчика (события бога проходят валидацию,
≥1 осмысленное событие/день); NPC упоминают реальные события недели; ИИ-поручения
выдаются и проходят валидатор в 100% случаев (фаззинг-тест); стоимость LLM ≤ бюджета
(лимиты в `ai/cost.py` под нагрузкой N игроков — посчитать при 100/1000 DAU).

   Это единственный способ подтвердить «10/10 затягиваемости» — симуляции не чувствуют скуку.

4. **Деплой:** Docker + VPS + systemd + ежедневный бэкап Postgres. Прод-бот без прокси.
