# Разбор референс-движка TeleMud (Go) — что берём в «Семь Корон»

Папка `TeleMud/` в корне — параллельная реализация Telegram-MUD на **Go**, заточенная под
огромный масштаб (5000 локаций, 10 000 NPC) и гибридный ИИ. Это «как сделать ту же игру под
сотни тысяч игроков». Наша «Семь Корон» — рукотворная, контентно-богатая, на Python/aiogram.
Поэтому инфраструктуру оттуда не берём, а ряд концепций — да.

## Берём сразу (дёшево, чистый Python)
1. **Utility AI для рядовых NPC.** Потребности `hunger/fatigue/fear/boredom(+greed)` ∈ [0,1],
   растут со временем; цель выбирается по кривым полезности (экспонента — страх; сигмоида — сон;
   логарифм — жадность) + **softmax с температурой** против «роботности».
   → реализовано в `engine/npc_ai.py`.
2. **FSM состояний NPC** (`IDLE/PATROL/COMBAT/FLEE/REST`): жёсткие переходы по событиям
   (получил урон → COMBAT; HP<20% и страх>0.6 → FLEE). → `engine/npc_ai.step_fsm`.
3. **Кольцевой буфер памяти NPC** (20 событий). → `MobAI.memory` / `remember()`.

## Адаптировать для нашего AI-слоя (`ai/`)
4. **Структура промпта LLM:** `[SYSTEM] [WORLD STATE] [RELEVANT MEMORIES] [DIALOGUE HISTORY]`,
   и LLM возвращает в конце JSON-действие `{"action":"give_quest","quest_id":...}`, которое
   исполняет движок. Превращает болтовню в реальные игровые действия.
5. **Контроль стоимости LLM** (прямо под наши проблемы с DeepSeek): семантический кэш
   (cos>0.95 → ответ из кэша), token-bucket на NPC/сутки, тайм-аут диалога 5 мин → сжать
   историю в одно «воспоминание».

## Позже, в урезанном виде
6. **Ленивая симуляция (Catch-up / delta-time):** при входе в локацию прокрутить
   `ΔT = now − last_tick` крупными квантами (10с/5мин/30мин). Обобщение нашего
   `process_respawns`/decay. Слой `npc_ai` уже считает потребности лениво по дельте.
7. **Процедурный генератор зон** (2D random-walk + BFS-связность + seed=`hash(region_id)`)
   для дешёвой набивки «диких» окраин вокруг рукотворного ядра.

## НЕ берём
Микросервисы, NATS, Redis Cluster, Qdrant, actor-модель, партиционирование — это под сотни
тысяч игроков; для одного aiogram-процесса оверинжиниринг. Их масштаб 5000 процедурных локаций
конфликтует с нашей силой — живым рукотворным лором и связанными квестами.

Подтверждают наш курс: гибридный UI (inline+reply+алиасы), батчинг боя через `editMessageText`,
MarkdownV2 — ровно их рекомендации по обходу лимитов Telegram.

## Что уже сделано в этой итерации
Реализован слой **#1–3** за флагом, не трогая боёвку:

- `engine/npc_ai.py` — Utility AI + FSM + ленивые потребности + ambient-тик (мобы вне боя
  бродят/дремлют; раненый в страхе моб сбрасывает агро и убегает в соседнюю комнату).
- Интеграция в `engine/loop.py` (`tick_ambient` под `if npc_ai.ENABLED`).
- Включение: переменная окружения **`NPC_AI=1`** (по умолчанию выключено), как у `RULES_V2`.
- Тесты: `test_npc_ai.py` (8 тестов). Существующие 278 тестов проходят, поведение по
  умолчанию не изменилось.

Слот `_ai` добавлен в `MobInstance.__slots__` для хранения состояния ИИ на экземпляре.

## Итерация 2: контроль стоимости LLM + действия NPC (#4–5)
Реализовано в `ai/`, активно автоматически когда включён ИИ-провайдер (флаг не нужен):

- `ai/cost.py`:
  - **SemanticCache** — кэш реплик по лексическому сходству вопроса (Jaccard, без
    эмбеддингов — работает офлайн). Похожий вопрос → ответ без обращения к модели.
  - **TokenBucket** — суточный лимит обращений на пару (игрок, NPC), сброс по дням.
  - **DialogueSession** — тайм-аут диалога: после паузы история сжимается локально в одно
    «воспоминание» и подставляется в следующий промпт (преемственность + экономия токенов).
- `ai/actions.py` — модель может в конце реплики вернуть JSON `{"action":"offer_quest",
  "quest_id":...}`; движок вырезает и **валидирует** по белому списку и реальной доступности
  (квест засчитывается только если он реально доступен у этого NPC). Выдуманные действия/id
  отбрасываются. Действие НЕ исполняется само — отдаётся боту как подсказка/кнопка.
- `ai/npc_ai.py` — новая `say_action()` (текст + валидированное действие); старая `say()`
  сохранена как обёртка (обратная совместимость). В system-промпт добавлены блок памяти и
  список разрешённых действий.

Настройки (env): `AI_DAILY_PER_NPC` (30), `AI_CACHE_SIM` (0.9), `AI_DIALOG_TIMEOUT` (300).
Тесты: `test_ai_cost.py` (5 шт). Все 278 тестов движка проходят.

**UI-кнопка действия (сделано):** оба места диалога в `bot/main.py` теперь зовут
`say_action()` и передают валидированное действие в `ui.kb_npc(ch, npc, highlight=act)`.
Новый хелпер `ui._npc_highlight_row` рисует заметную кнопку-призыв над обычными:
`offer_quest` → «✨📜 Взять задание: …» (callback `qaccept:<qid>`), `to_vendor` →
«✨🛒 Показать товары» (`shop:<npc>`), `to_trainer` → «✨🎓 К обучению» (`train`).
Кнопки переиспользуют существующие колбэки — новых обработчиков не нужно.

## Итерация 3: процедурный генератор «диких» зон (#7)
`engine/worldgen.py` — связный граф комнат в формате WORLD: 2D random-walk + **остовное
дерево** (гарантирует связность) + добавочные рёбра-петли + BFS-проверка. Детерминирован по
seed (`_derive_seed` = base XOR FNV-хэш(region_id)). `attach()` подвешивает зону к комнате-
якорю, **не затирая** рукотворный контент; `apply_wild_zones()` цепляет набор готовых зон
(`WILD_ZONE_SPECS`) к диким комнатам.

Включение: env **`WILD_ZONES=1`** (+ опц. `WILD_SEED`); по умолчанию выключено — мир ровно
тот же (92 комнаты). С флагом: +54 комнаты (2 зоны), все достижимы из village, `validate()`
проходит. Предпросмотр: `scripts/gen_wild_preview.py` (ASCII-карта). Тесты: `test_worldgen.py`
(7 шт: размер/схема, связность, двусторонние выходы, детерминизм, attach-безопасность,
целостность мира). Гайд — `Instruction/Дикие_зоны.md`.

Назначение — дёшево набивать дикие окраины вокруг handcrafted-ядра, НЕ заменяя его
(наша сила — рукотворный лор и связанные квесты).

## Итерация 4: ленивая Catch-up симуляция (#6)
`engine/catchup.py` — комнаты делятся на активные (игроки + соседи 1-го радиуса, каскадная
преактивация) и спящие. Спящие не тикаются; при входе игрока комната «догоняется»: респавн
мобов по абсолютному `dead_at` (корректно при любом ΔT) + рост потребностей NPC закрытой
формой (`npc_ai.update_needs`), ΔT бьётся на крупные кванты 10с/5мин/30мин (`coarse_steps`).

Включение: env **`LAZY_SIM=1`**; по умолчанию выключено. Хук в `engine/loop.py.tick`: при
флаге combat/ambient идут только по активным комнатам, а `catchup.tick()` заменяет глобальный
`process_respawns` (ленивый респавн при активации). Бой и так лишь там, где игроки → для
игрока поведение не меняется, экономится холостая работа. Тесты `test_catchup.py` (6) +
поведенческий тест обоих режимов (OFF=глобальный респавн; ON=спящая комната оживает на входе).
Гайд — `Instruction/Ленивая_симуляция.md`.

## Итог по референсу
Взято и внедрено (всё за флагами, дефолт не меняет игру): Utility AI + FSM живых NPC
(`NPC_AI`), контроль стоимости LLM + действия NPC (авто), процедурный генератор зон
(`WILD_ZONES`), ленивая Catch-up симуляция (`LAZY_SIM`). НЕ берём: микросервисы, NATS, Redis
Cluster, Qdrant, actor-модель — оверинжиниринг под наш масштаб (один aiogram-процесс).
