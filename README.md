# 🗡 СЕМЬ КОРОН v3 — фэнтези-MUD-ядро для Telegram

Полноценное MUD-ядро: **6 классов и 5 рас со своими характеристиками и умениями**, **реал-тайм бой** (мобы атакуют сами по таймеру, как в классических MUD), **кооператив** (несколько игроков бьют одного моба, делят опыт, жрец лечит союзников), **экономика** (магазин, продажа добычи, крафт у кузнеца), **квесты**, **PostgreSQL** для хранения, и мир из 17 локаций с тремя ветками и финальным боссом. Весь контент вынесен в **YAML** — мир расширяется без правки кода.

## Что нового по сравнению с v2

| | v2 | v3 |
|---|---|---|
| Классы/расы | нет | **6 классов + 5 рас** с атрибутами str/dex/int/spi и ограничениями |
| Умения | нет | **18 скиллов** с маной, кулдауном, эффектами (щит, яд, уклонение, лечение, вампиризм) |
| Бой | пошаговый по кнопке | **реал-тайм тики** — мобы бьют по своим таймерам |
| Хранилище | JSON-файл | **PostgreSQL** (asyncpg), JSON-fallback при отсутствии БД |
| Мир | 8 комнат в коде | **17 комнат в YAML**, 15 мобов, лут, NPC, отдых в храме |
| Экономика | магазин | + **продажа добычи** и **крафт у кузнеца** (рецепты в YAML) |
| Квесты | нет | **9 квестов** (kill/collect, цепочки, награды, журнал) |
| Кооп | общие HP мобов | + **деление опыта/лута**, лечение союзников, агро-система |

## Структура проекта

```
mud2/
├── data/              ← КОНТЕНТ (правится без кода)
│   ├── classes.yaml   ← 6 классов: атрибуты, рост, умения
│   ├── races.yaml     ← 5 рас: модификаторы, пассивки, ограничения классов
│   ├── skills.yaml    ← 18 умений: урон/лечение/баффы, мана, кулдаун
│   ├── world.yaml     ← 17 комнат: выходы, спавны, NPC, предметы
│   ├── mobs.yaml      ← 15 врагов: статы, лут, респавн, скорость атаки
│   ├── items.yaml     ← оружие, броня, аксессуары, расходники, лут
│   ├── quests.yaml    ← 9 квестов: цели, награды, цепочки
│   └── recipes.yaml   ← рецепты крафта у кузнеца
├── engine/            ← ЯДРО (механика, не зависит от Telegram)
│   ├── content.py     ← загрузка + валидация YAML, цены продажи
│   ├── character.py   ← модель героя, расчёт характеристик
│   ├── world.py       ← рантайм мобов, респавн
│   ├── combat.py      ← формулы урона, скиллы, эффекты, кулдауны
│   ├── loop.py        ← реал-тайм игровой цикл (тики)
│   ├── quest.py       ← приём/прогресс/сдача квестов
│   ├── craft.py       ← крафт предметов из материалов
│   ├── social.py      ← пати и дуэли (написаны, к боту не подключены)
│   └── db.py          ← PostgreSQL-слой
├── bot/               ← TELEGRAM
│   ├── ui.py          ← рендеры + inline-клавиатуры
│   └── main.py        ← хендлеры, точка входа
├── test_engine.py     ← автономные тесты движка (без Telegram/БД)

```

Это **слоистая архитектура**: `engine/` ничего не знает про Telegram (его можно протестировать или подключить к другому интерфейсу — веб, Discord), `bot/` — тонкий слой представления.

## Установка и запуск

### 1. Зависимости
Требуется **Python 3.12+**.
```bash
pip install -r requirements.txt -c constraints.txt
```
`constraints.txt` фиксирует проверенные точные версии зависимостей (см. файл).
Диапазоны версий по-прежнему заданы в `requirements.txt` — `constraints.txt`
их не расширяет, только сужает до конкретных релизов.

### 2. PostgreSQL на VPS
```bash
sudo apt install postgresql
sudo -u postgres psql -c "CREATE DATABASE mud;"
sudo -u postgres psql -c "CREATE USER muduser WITH PASSWORD 'ВСТАВЬ_СВОЙ_ПАРОЛЬ';"
sudo -u postgres psql -c "GRANT ALL ON DATABASE mud TO muduser;"
```
Таблицы создаются автоматически при первом запуске.

### 3. Переменные окружения и запуск
```bash
export BOT_TOKEN="токен_от_BotFather"
export DATABASE_URL="postgresql://muduser:ВСТАВЬ_СВОЙ_ПАРОЛЬ@localhost/mud"
cd mud2
python -m bot.main
```

> Если `DATABASE_URL` недоступен, игра **всё равно запустится** без сохранения (удобно для локальных тестов) — в консоли будет предупреждение.

## Геймплей

1. `/start` → выбор **расы** → выбор **класса** → ввод имени.
2. Дальше всё **кнопками**: атака, умения, перемещение, сумка, магазин, квесты.
3. В бою мобы бьют **сами** по таймеру — лечитесь и бейте в ответ.
4. Умри — очнёшься в **Святилище** (запад от деревни), там же полное восстановление.
5. Три пути к руинам: **лес** (север), **рудники** (восток, там кузнец-магазин/крафт), **болото** (юг). Все сходятся у **Врат**, за ними — **Падший Король** (480 HP, зовите друзей в кооп).

Текстовые команды-помощники: `сказать <текст>` (чат комнаты), `кто` (онлайн), `купить <предмет>`, направления словами.

## Классы

- **⚔️ Воин** — много HP, урон от Силы, −20% получаемого урона. Умения: Мощный удар, Стена щитов, Рассекающий удар.
- **🔮 Маг** — большой бурст от Интеллекта, мало HP, зависит от маны. Огненный шар, Ледяная броня, Чародейский залп.
- **🗡 Разбойник** — урон от Ловкости, +15% крит, 25% двойной удар. Удар в спину, Уклонение, Отравленный клинок.
- **✨ Жрец** — лечит себя и **союзников в комнате** (ключевой для кооп), урон от Духа. Исцеление, Кара, Благословение.
- **🛡 Паладин** — гибрид: броня + урон от Силы + самолечение и AoE-лечение. Святой удар, Возложение рук, Освящение.
- **💀 Некромант** — урон от Интеллекта + **вампиризм** (возврат 25% урона в HP), хрупкий. Высасывание жизни, Костяная броня, Проклятие гнили.

**Расы** (реально влияют на билд): 🧑 Человек (+опыт), 🧌 Орк (+HP/Сила, без мага), 🧝 Эльф (крит/уворот, хрупкий), 🧔 Гном (прочность/золото), 👺 Гоблин (уворот/удача, только маг/разбойник/некромант).

## Экономика: магазин, продажа, крафт

У кузнеца (рудники) три вкладки:

- **🛒 Покупка** — зелья, оружие, броня, аксессуары.
- **💰 Продажа добычи** — скупка материалов и ненужного снаряжения за ~60% цены (надетое не продаётся). Даёт смысл всему луту.
- **🔨 Ковка** — крафт снаряжения из материалов + золото по рецептам из `recipes.yaml` (кольчуга, стальной меч, латы, кристальный амулет).

## Прогрессия и реморт

- **Уровни 1–60** — прогрессия ограничена капом 60-го уровня; классовые умения изучаются в этом диапазоне (последние капстоуны — на 55–60).
- **Таланты** — очко даётся раз в 4 уровня (уровни 4, 8, …, 60) → ровно **15 очков** к капу; сумма рангов талантов на класс = 15.
- **Реморт-престиж** — на капе доступно перерождение: круг реморта даёт навсегда **+5% к силе и здоровью** (потолок **+50%** на 10-м круге); уровень и снаряжение сбрасываются, бонус копится.
- **Эндгейм для перерождённых** — **46 реморт-предметов** (бывшие ур.75/90, теперь level_req 56–60) требуют 1–2 перерождения (⭐), не выпадают и не надеваются без реморта; финальный босс Предвечной Бездны (ур.68) рассчитан на слаженную группу или перерождённых.

## Флаги мира

Из коробки (`python -m bot.main`) включены семь флагов «живого мира»: `NPC_AI` (NPC бродят/отдыхают вне боя), `LAZY_SIM` (докрутка мира за время оффлайна игрока), `WILD_ZONES` (дикие зоны вокруг основного мира), `WORLD_EVENTS` (мировые события по таймеру), `SEASONS` (сезонный трек/лиги), `NOTIFY` (уведомления игроку) и `RULES_V2` (гибридное ядро правил: типы урона, резисты, спасброски, мультиатака). Каждый можно отключить переменной окружения или в `.env`, например `NPC_AI=0`. `RULES_V2` включён по умолчанию с спринта 7: мультиатака после балансировки (`multiattack_scale`, спринт 6) укладывается в целевой коридор ±25% ΔDPS для всех 6 классов (Monte-Carlo N=350×30 раундов, см. `sim_rules2.py`); последний блокер — профиль мобов `spirit`, резистивший сразу все физические типы урона — снят решением лида: дробящее (bash — освящённая булава/посох) сокрушает бесплотную форму и проходит, режущее/колющее (pierce/slash) по-прежнему резистится. Против undead/demon/default все 6 классов в коридоре ±25%; против spirit классы с дробящим оружием (priest/paladin) и воин тоже в коридоре, mage/rogue/necromancer (кинжал/лук) осознанно проседают — контрплей у всех классов даёт 100% боевых скиллов, уже нефизических по типу урона. Полный список и обоснование дефолтов — в `.env.example`.

## Как расширять мир

Добавить комнату — допиши блок в `world.yaml` и пропиши выход из существующей. Новый моб — `mobs.yaml` + добавь его id в `spawns` комнаты. Новый предмет — `items.yaml`. При запуске `engine/content.py` валидирует ссылочную целостность (несуществующие выходы/мобы/лут — ошибка с указанием места):
```bash
python -m engine.content   # проверка контента
```

## Заметки по продакшену

- Состояние мобов (HP, респавн, агро) живёт в памяти процесса — при рестарте бот возрождает мир из YAML. Персонажи персистятся в Postgres.
- Игровой цикл тикает раз в секунду одной фоновой задачей; масштабируется до сотен игроков на одном процессе. Для тысяч — шардировать по зонам.
- Сохранение идёт после каждого значимого действия; для высокой нагрузки стоит добавить дебаунс/батчинг записи.
